Introduction

Go has earned its place as a top choice for building backend services. Its simplicity, excellent standard library, built-in concurrency, and fast compilation make it ideal for production APIs that need to handle thousands of requests per second with minimal resource usage.

In this guide, I'll walk through building a complete production-ready API from scratch, covering everything from project structure to deployment. These patterns come from real-world experience shipping Go services that handle millions of requests daily.

Why Go for APIs?

Before diving in, let's understand why Go stands out for API development:

  • Performance â€” Compiled to native code, Go APIs typically respond in microseconds, not milliseconds
  • Concurrency â€” Goroutines handle thousands of concurrent connections with minimal memory overhead
  • Standard library â€” net/http is production-grade out of the box, no framework required
  • Single binary â€” Deploy a single executable, no runtime dependencies
  • Fast builds â€” Compile a large project in seconds, not minutes

Server Setup with Graceful Shutdown

A production server must handle graceful shutdown â€” finishing in-flight requests before stopping. Here's the pattern I use in every project:

func main() {
    cfg := config.Load()

    // Initialize database
    db, err := database.Connect(cfg.DatabaseURL)
    if err != nil {
        log.Fatalf("Failed to connect to database: %v", err)
    }
    defer db.Close()

    // Initialize dependencies
    userRepo := repository.NewUserRepository(db)
    userService := service.NewUserService(userRepo)
    userHandler := handler.NewUserHandler(userService)

    // Setup router
    mux := http.NewServeMux()
    mux.HandleFunc("GET /api/users", userHandler.List)
    mux.HandleFunc("GET /api/users/{id}", userHandler.Get)
    mux.HandleFunc("POST /api/users", userHandler.Create)

    // Apply middleware stack
    stack := middleware.Chain(
        middleware.Logger,
        middleware.CORS,
        middleware.RateLimit(100),
        middleware.Recover,
    )

    // Create server
    srv := &http.Server{
        Addr:         fmt.Sprintf(":%d", cfg.Port),
        Handler:      stack(mux),
        ReadTimeout:  15 * time.Second,
        WriteTimeout: 15 * time.Second,
        IdleTimeout:  60 * time.Second,
    }

    // Start server in goroutine
    go func() {
        log.Printf("Server starting on :%d", cfg.Port)
        if err := srv.ListenAndServe(); err != http.ErrServerClosed {
            log.Fatalf("Server error: %v", err)
        }
    }()

    // Wait for interrupt signal
    quit := make(chan os.Signal, 1)
    signal.Notify(quit, syscall.SIGINT, syscall.SIGTERM)
    <-quit

    log.Println("Shutting down server...")

    // Give in-flight requests 30 seconds to complete
    ctx, cancel := context.WithTimeout(context.Background(), 30*time.Second)
    defer cancel()

    if err := srv.Shutdown(ctx); err != nil {
        log.Fatalf("Server forced shutdown: %v", err)
    }

    log.Println("Server stopped gracefully")
}

Middleware Stack

Every production API needs proper middleware. Here's how I implement a composable middleware chain:

type Middleware func(http.Handler) http.Handler

func Chain(middlewares ...Middleware) Middleware {
    return func(next http.Handler) http.Handler {
        for i := len(middlewares) - 1; i >= 0; i-- {
            next = middlewares[i](next)
        }
        return next
    }
}

// Logging middleware
func Logger(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        start := time.Now()
        wrapped := &responseWriter{ResponseWriter: w, statusCode: 200}
        next.ServeHTTP(wrapped, r)
        log.Printf("%s %s %d %s",
            r.Method, r.URL.Path,
            wrapped.statusCode, time.Since(start),
        )
    })
}

// Recovery middleware
func Recover(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        defer func() {
            if err := recover(); err != nil {
                log.Printf("panic: %v\n%s", err, debug.Stack())
                http.Error(w, "Internal Server Error", 500)
            }
        }()
        next.ServeHTTP(w, r)
    })
}

Structured Error Handling

Use custom error types that map cleanly to HTTP status codes. This keeps your handlers clean and your error responses consistent:

type AppError struct {
    Code    int    `json:"-"`
    Message string `json:"message"`
    Detail  string `json:"detail,omitempty"`
}

func (e *AppError) Error() string { return e.Message }

var (
    ErrNotFound     = &AppError{Code: 404, Message: "Resource not found"}
    ErrUnauthorized = &AppError{Code: 401, Message: "Unauthorized"}
    ErrBadRequest   = &AppError{Code: 400, Message: "Invalid request"}
)

// Handler helper
func writeError(w http.ResponseWriter, err error) {
    var appErr *AppError
    if errors.As(err, &appErr) {
        writeJSON(w, appErr.Code, appErr)
    } else {
        writeJSON(w, 500, map[string]string{"message": "Internal server error"})
    }
}

Database Layer with SQLC

I use SQLC for type-safe database access. You write SQL, it generates Go code. No ORM overhead, full control over queries, and complete type safety:

-- queries/users.sql
-- name: GetUser :one
SELECT id, name, email, created_at
FROM users
WHERE id = $1;

-- name: ListUsers :many
SELECT id, name, email, created_at
FROM users
ORDER BY created_at DESC
LIMIT $1 OFFSET $2;

-- name: CreateUser :one
INSERT INTO users (name, email, password_hash)
VALUES ($1, $2, $3)
RETURNING id, name, email, created_at;

SQLC generates type-safe Go functions from these queries, catching SQL errors at build time rather than runtime.

Configuration Management

Use environment variables with sensible defaults. Never hardcode credentials:

type Config struct {
    Port        int
    DatabaseURL string
    JWTSecret   string
    Environment string
}

func Load() *Config {
    return &Config{
        Port:        getEnvInt("PORT", 8080),
        DatabaseURL: getEnv("DATABASE_URL", "postgres://localhost/myapp"),
        JWTSecret:   getEnv("JWT_SECRET", ""),
        Environment: getEnv("ENVIRONMENT", "development"),
    }
}

Production Checklist

Before deploying any Go API to production, ensure you have:

  • Graceful shutdown handling
  • Structured logging (JSON format for log aggregation)
  • Request/response timeouts configured
  • CORS middleware for frontend clients
  • Rate limiting to prevent abuse
  • Input validation on all endpoints
  • Health check endpoint (/healthz)
  • Database connection pooling configured
  • Panic recovery middleware
  • Environment-based configuration

Conclusion

Building production-ready Go APIs requires attention to error handling, logging, graceful shutdown, and proper project structure. The standard library gives you everything you need â€” no heavy frameworks required. The patterns shown here have served me well across multiple production deployments handling real traffic at scale.

Start simple, add complexity only when needed, and always write tests. Go makes all of this straightforward.
