Technical Architecture

How I Build Things

A look into the architecture and design patterns I use across projects.

Backend Architecture

My Go backends follow clean architecture principles with clear separation between handlers, services, and repositories. I use SQLC for type-safe database queries and implement proper error handling with custom error types.

// Example Go service structure
type UserService struct {
    repo    *repository.UserRepository
    cache   cache.Cache
    events  events.Publisher
}

func (s *UserService) CreateUser(ctx context.Context, req CreateUserRequest) (*User, error) {
    // Validation, business logic, and persistence
    user, err := s.repo.Create(ctx, req.ToModel())
    if err != nil {
        return nil, fmt.Errorf("create user: %w", err)
    }
    
    // Publish domain event
    s.events.Publish(ctx, UserCreatedEvent{UserID: user.ID})
    
    return user, nil
}

Mobile Architecture

Flutter apps use a feature-first folder structure with BLoC or Riverpod for state management. I implement repository patterns for data access and use dependency injection for testability.

Real-time Systems

For real-time features, I build on WebSockets with proper reconnection logic, message queuing, and presence management. Video/audio features use LiveKit or Agora SDKs with custom UI layers.

Infrastructure

All projects are containerized with Docker and deployed via CI/CD pipelines. I use PostgreSQL for relational data, Redis for caching and pub/sub, and proper logging/monitoring setups.
